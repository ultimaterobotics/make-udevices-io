<p class="rm-label">Hardware wishlist</p>

We maintain this list publicly so the community can see exactly what we need and why. Support us on [Open Collective](https://opencollective.com/udevices).

<div class="rm-card" style="padding: 4px 14px; margin-top: 0.75rem;">
  <div class="hw-item">
    <div class="hw-header">
      <span class="hw-name">Raspberry Pi 4 or 5</span>
      <span class="hw-cost">~$70</span>
      <span class="badge badge-next">next</span>
    </div>
    <p class="hw-why">Most requested platform — BLE via Bleak, embedded use cases, robotics integration</p>
  </div>
  <div class="hw-item">
    <div class="hw-header">
      <span class="hw-name">Android test devices × 2–3</span>
      <span class="hw-cost">~$120</span>
      <span class="badge badge-next">next</span>
    </div>
    <p class="hw-why">BLE compatibility varies by chipset and Android version — secondhand devices needed to test reliably across hardware</p>
  </div>
  <div class="hw-item">
    <div class="hw-header">
      <span class="hw-name">Raspberry Pi CM5</span>
      <span class="hw-cost">~$90</span>
      <span class="badge badge-next">next</span>
    </div>
    <p class="hw-why">Embedded platform support — compact deployments, wearable integration, standalone sensor nodes</p>
  </div>
  <div class="hw-item">
    <div class="hw-header">
      <span class="hw-name">iPhone + Mac mini</span>
      <span class="hw-cost">~$573</span>
      <span class="badge badge-later">later</span>
    </div>
    <p class="hw-why">iOS BLE development requires physical Apple hardware — unlocks support for a large segment of users currently unable to use the app</p>
  </div>
</div>
